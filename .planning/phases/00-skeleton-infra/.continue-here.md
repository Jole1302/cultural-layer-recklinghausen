## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| vanilla-cookieconsent v3 ESM-CJS interop | Library publishes CJS; ESM `import()` exposes API on `.default`, not top-level. TypeScript types claim top-level. | advisory | Use `((mod as unknown) as { default: typeof mod }).default ?? mod` cast pattern (see src/components/cookie-banner.tsx) |
| `'server-only'` import breaks vitest | `'server-only'` package throws when imported outside React Server Components context; vitest runs in plain Node. | advisory | vitest.config.ts has `resolve.alias['server-only'] = './tests/setup/server-only-stub.ts'` |
| Vitest 4 poolOptions removed from `test` | Vitest 4 moved poolOptions out of `test:`; type defs don't accept it top-level either. | advisory | Removed entirely; default `singleFork: false` is what we want |
| `vercel env add NAME preview` requires git | Vercel CLI needs a connected Git repo before per-branch preview env vars can be added. | advisory | Defer preview env vars until T-30 (GitHub repo + Vercel Git connection) |
| Drizzle wraps pg errors | `await db.insert(...).rejects.toThrow(/regex/)` matches the wrapped Drizzle error message, not the original pg error. | advisory | Use `try/catch` and inspect `err.cause.code` (e.g. `'23505'` for unique violation) and `err.cause.constraint` (see tests/unit/audit.test.ts) |
| `vercel blob create-store` interactive prompts | The CLI prompts for env-multi-select via arrow keys; piped stdin doesn't work. | advisory | Use `--yes` flag — it auto-selects all envs when paired with `--access` |
| `.env.local` double-quotes (Vercel CLI format) | `vercel env pull` emits `KEY="value"` with double-quotes; plain `string.trim()` does NOT strip them. | advisory | tests/setup/load-env.ts strips surrounding double-quotes; production Next.js parses them correctly via dotenv |
| Playwright e2e in vitest runner | Without exclude, vitest picks up tests/e2e/*.spec.ts and crashes with "Playwright Test did not expect test() to be called here" | advisory | vitest.config.ts has `exclude: ['**/node_modules/**', '**/tests/e2e/**']` |
| `@sentry/nextjs` v10 removed `hideSourceMaps` | SentryBuildOptions v10.50.0 no longer has `hideSourceMaps`; use `sourcemaps` object instead. | advisory | Removed from next.config.ts; widenClientFileUpload kept (still valid) |
| Better Auth tables need manual pg migration | `drizzle-kit migrate` reads config `schema:` which points only to domain schema; Better Auth SQL must be applied via `pg.Pool.query()` directly or via a separate drizzle.config.ts. | advisory | Applied manually via node + pg Pool in T-25; journal updated to include 0001_better_auth entry |
